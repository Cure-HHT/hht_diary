## REQ-d00115 assertions (6)

A. The system SHALL append exactly one `ProvenanceEntry` to `event.metadata.provenance` on each hop that receives the event.
B. The system SHALL NOT mutate any `ProvenanceEntry` already present in the chain.
C. Each `ProvenanceEntry` SHALL carry the fields `hop`, `received_at`, `identifier`, `software_version`, and optional `transform_version`.
D. The `identifier` SHALL be a device UUID for mobile-device hops; server instance identifier for server hops.
E. The `software_version` SHALL follow the format `"<package-name>@<semver>[+<build>]"`.
F. The `transform_version` SHALL be non-null iff the incoming wire payload was produced by a transform at the previous hop.

## REQ-d00116 assertions (7)

A. Carries an `id` string matching `event.entry_type` for every event of the type.
B. Carries a `version` string identifying the schema version.
C. Carries a `name` string for display.
D. Carries a `widget_id` string keyed into the Flutter widget registry.
E. Carries a `widget_config` JSON payload whose shape is determined by the widget.
F. MAY carry a nullable `effective_date_path` JSON path usable by the materializer.
G. MAY carry an optional `destination_tags` list of strings.
