The demo configuration triggered an existing latent bug: `runHistoricalReplay` called `destination.transform(batch)` unconditionally, but native destinations (`serializesNatively == true`) throw on `transform` by contract (REQ-d00152-B). Adding `NativeAudit` with `includeSystemEvents: true` exposed the bug because every prior bootstrap-emitted `system.entry_type_registry_initialized` and `system.destination_registered` audit now matches `NativeAudit`'s filter and runs through replay during `setStartDate`.

Fix: `runHistoricalReplay` now branches on `destination.serializesNatively` symmetrically with `fillBatch`. When native, replay mints a `BatchEnvelopeMetadata` from the caller-supplied `Source` and enqueues via `nativeEnvelope:`; when non-native, it preserves the existing `transform` + `wirePayload` path. `runHistoricalReplay` accepts a new optional `source:` parameter (defaulted to `null`); calling it without `source` on a native destination raises `ArgumentError`. `DestinationRegistry.setStartDate` (the sole caller) threads `_eventStore.source` through.
